### توضیحات:

 - تمامی رنگ ها براساس صفحه colors در دیزاین سیستم تعریف شده اند

 - از رنگ های کاستوم و خارج از مواردی که تعریف شده نمیتوانید استفاده کنید

 - به رنگ های تیلویند نمیتوانید رنگ اضافه کنید مگر در صفحه colors دیزاین سیستم اضافه شده است

 - اگر رنگی در طراحی ها وجود داشته باشد که خارج از پالت موجود بود دو راه وجود دارد
    - به نزدیک ترین رنگی که در پالت موجود بود تغییر دهید
    - به سرپرست قسمت خود اطلاع دهید تا این موضوع را پیگیری کند

### شیوه ی نامگذاری رنگ ها:

اسم رنگ ها بر اساس بیشترین استفاده ای که آن رنگ در دیزاین ها، در زمان طراحی دیزاین سیستم داشته، انتخاب شده است.

پس اسم رنگ ها مربوط به کاربرد آن ها در دیزاین است. یعنی عمدتا رنگ هایی داریم که مربوط به مشخصه های خاصی از کامپوننت های مختلف می شود. علاوه بر آن رنگ های primary و secondary و همین طور رنگ هایی مربوط به استیت های خاص نیز داریم.

حتما رنگ هایی خواهند بود که با گذر زمان به آن ها نیاز پیدا خواهیم کرد و به این پالت اضافه خواهد شد و براساس آن داکیومنت کانفلوينس آن نیز باید آپدیت شود.

### شیوه ی استفاده از رنگ ها:

برای استفاده از رنگ ها دقیقا طبق داکیومنت tailwind باید عمل کرد مثلا:

#### background :
```jsx
  <div className="bg-primary-description" />
```
#### text color :
```jsx
  <div className="text-error" />
```
#### border color :
```jsx
  <div className="border-idle-border" />
  <div className="border-success" />
```
#### hover & focus & ... :
```jsx
  <div className="hover:bg-success-hover" />
```

رنگ هایی که داخل آبجکت نوشته می شوند در className برای استفاده از آنها باید از - یا dash استفاده کرد.

رنگ های nested ای که نام آن ها DEFAULT است برای استفاده از آنها کافیست اسم parent آنها را به کار برد مثلا :

bg-primary به جای bg-primary-DEFAULT

در صورت اضافه کردن رنگ حتما داکیومنت را آپدیت کنید.
