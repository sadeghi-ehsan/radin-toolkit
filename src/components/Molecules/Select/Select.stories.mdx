### شماره تسک در جیرا:

### توضیحات:

برای کدنویسی کامپوننت Select از پکیج‌های متعددی استفاده شده است که مهم‌ترین آن‌ها [react-select](https://react-select.com/home "react-select") می‌باشد. کامپوننت Select از ۵ نوع مختلف تشکیل شده است که هرکدام از کامپوننت‌های مختلفی استفاده می‌کنند:
  - default: react-select
  - async: react-select/async
  - creatable: react-select/creatable
  - asyncPaginate: react-select-async-paginate

نکته‌ای دیگر این است که از آنجایی که کامپوننت Select زمانی استفاده می‌شود که ممکن است لیست‌های بلند بالا داشته باشیم، از پکیج دیگری به نام react-select-virtualized استفاده شده است تا به صورت مجازی لیست‌ها لود شوند و سرعت رندر بسیار بالا‌تر خواهد رفت.

### props ها و قابلیت ها:

تمامی propsها به طور هوشمند پیاده‌سازی شده‌اند، یعنی متناسب با typeای که انتخاب می‌کنید، propsهای همان type به شما پیشنهاد داده می‌شوند. یعنی اگر default انتخاب شده باشد، دیگر props های به نام loadOptions نخواهید داشت ولی اگر async انتخاب شده باشد، این props وجود دارد.

- **id**
- **onFieldsetClick** تابعی که با کلیک بر روی fieldset صدا زده می‌شود. برای زمانی کاربرد دارد که می‌خواهیم با کلیک بر روی هر قسمت از fieldset، المان input را focus کنیم.
- **onClear** تابعی که با کلیک بر روی دکمه ی ضربدر مقدار داخلی اینپوت را خالی میکند و در صورت پاس ندادن آن به رپر دکمه ی ضربدر را نخواهیم داشت حتی اگر clearable مقداری صحیح باشد.
- **clearable** بولین است و با true بودن آن رپر دارای دکمه ی پاک کردن مقدار اینپوت می شود ( هنوز نیاز به وجود کال بک  onClear داریم)
- **allowClear** مشخص می‌کند که آیا آیکون clear فعال و قابل کلیک باشد یا خیر. زمانی که مقدار input خالی باشد، آیکون نمایش داده می‌شود ولی دیگر قابل کلیک نخواهد بود
- **preserveErrorBox** بولین است که در صورت true بودن، چه error داشته باشیم و چه نه، فضای مورد نیاز برای نمایش error اشغال خواهد شد.
  - این کار به جهت آن است که در فرم‌ها زمانی که خطا نمایش داده می‌شود، پرش در فرم نداشته باشیم
- **type**
  -default
    - حالت پیش فرض کامپوننت میباشد برای مواقعی که دیتای کمی وجود دارد و در واقع یک ابجکت ساده میباشد
  -async
    - برای زمانی استفاده میشود که میخواهیم دیتایی از سمت سرور دریافت کنیم
  -creatable
    - برای زمانی استفاده میشود که میخواهیم اگر دیتای مورد نیاز در لیست نبود آن را ایجاد کنیم
  -asyncCreatable
    - برای زمانی استفاده میشود که میخواهیم اگر دیتای در سمت سرور نبود آن را ایجاد کنیم
  -asyncPaginate
    - برای api هایی که به صورت صفحه بندی شده دیتارا به ما ارسال میکنند

- **searchable** امکان جستجو به کاربر میدهد
- **loading** اگر فعال باشد آیکون لودینگ اضافه میشود
- **multi** به صورت دیفال غیر فعال است اگر فعال باشد قابلیت چند انتخابی به شما میدهد
- **placeholder** متنی که در پس‌زمینه Select نمایش داده می‌شود

و مابقی propsها رو طبق راهنمای سایت [react-select](https://react-select.com/home "react-select") و [react-select-async-paginate](https://github.com/vtaits/react-select-async-paginate "react-select-async-paginate") متناسب با کاربردی که مدنظر هست، بخوانید و استفاده کنید.
