### شماره تسک در جیرا:

### توضیحات:

کامپوننت **Input** از کامپوننت‌های FormElementWrapper و BaseInput استفاده می‌کند. به طوری که تمام propsهای آن‌ها نیز در کامپوننت input هم موجود هستند


### props ها و قابلیت ها:

- **id**
- **children** کامپوننت‌هایی که قرار است از این wrapper استفاده کنند
- **label** عنوان

- **variant** حالت های متفاوت طراحی و پیاده شده :
  - line
  - outline
  - fill
  - none

- **startAdornment** می تواند Icon , badge یا هر نوع ReactNode باشد.

- **endAdornment** می تواند Icon , badge یا هر نوع ReactNode باشد.

- **clearable** بولین است و با true بودن آن رپر دارای دکمه ی پاک کردن مقدار اینپوت می شود ( هنوز نیاز به وجود کال بک  onClear داریم)
- **allowClear** مشخص می‌کند که آیا آیکون clear فعال و قابل کلیک باشد یا خیر. زمانی که مقدار input خالی باشد، آیکون نمایش داده می‌شود ولی دیگر قابل کلیک نخواهد بود
- **important** که بولین است و با true بودن آن دیزاین رپر طلایی می شود
- **required** که بولین است و با true بودن یک بولت و ولیدیشن به آن اضافه میشود
- **size** با مشخص کردن این props، ارتفاع wrapper تغییر خواهد کرد
  -sm
  -md
  -lg
- **legendClassName** کلاسی که به legend داده میشود.
- **fieldsetClassName** کلاسی که به fieldset داده میشود.
- **childrenClassName** کلاسی که در رپر به children آن داده میشود. پس هیچ کلاسی در بیرون از رپر به این children نباید داد چرا که این کلاسی از بیرون داده ایم override خواهد شد.
- **verified** بولین است که در صورت true بودن تیک سبز نشان داده خواهد شد.
- **error** می تواند سه حالت داشته باشد:
  - اگر true باشد آنگاه فقط بردر FormElementWrapper به رنگ ارور قرار میگیرد.
  -  اگر از تایپ string باشد آنگاه بردر قرمز و دارای ارور مسیج خواهد بود.
  - اگر false باشد آنگاه نه بردر دارد و نه ارور مسیج.
- **help** می تواند متن یا هر نوع ReactNode باشد. این متن زیر fieldset نمایش داده خواهد شد
- **disabled** بولین است که در صورت true بودن نحوه نمایش تغییر خواهد کرد و خاکستری می‌شود و کامپوننت Input دیگر قابل استفاده نخواهد بود
  - در این حالت، آیکون clear هم نمایش داده نخواهد شد
- **block** بولین است که در صورت true بودن کامپوننت wrapper تمام خط را اشغال خواهد کرد
  - مقدار پیش‌فرض true می‌باشد
- **preserveErrorBox** بولین است که در صورت true بودن، چه error داشته باشیم و چه نه، فضای مورد نیاز برای نمایش error اشغال خواهد شد.
  - این کار به جهت آن است که در فرم‌ها زمانی که خطا نمایش داده می‌شود، پرش در فرم نداشته باشیم
- **value** اگر بخواهیم input به صورت uncontrolled باشد، باید value آن را undefined پاس دهیم یا در کل پاس ندهیم
  - در غیر این‌صورت باید مقادیر value و onChange را تنظیم کنیم تا input از نوع Controlled باشد
- **onChange** تابعی است که هنگام تغییر در مقدار input صدا زده می‌شود و به ترتیب value و name و event را برمیگرداند
- **name** متنی که مشخص کننده مقدار name مربوط به تگ input می‌باشد
  - این props به صورت اجباری هنگام استفاده از کامپوننت باید تنظیم شود
- **readOnly** بولین است که در صورت true بودن، دیگر امکان تغییر مقدار value نمی‌باشد و کاربر نمی‌تواند چیزی تایپ کند ولی شکل ظاهری تغییری نخواهد کرد
- **placeholder** متنی که در پس‌زمینه input نمایش داده می‌شود
- **autoComplete** متنی که مربوط به تگ input می‌باشد و جهت کنترل کردن نمایش پیشنهادات هنگام تایپ توسط مرورگر می‌باشد
- **align** مشخص کننده چپ یا راست یا وسط چین بودن Input می‌باشد
  - left
  - right
  - center
- **baseInputProps** گاهی نیاز است تا به صورت مستقیم یکسری attributes برای تگ input تنظیم کنیم. که در این حالت می‌توان یک آبجکت به این props پاس دهیم
