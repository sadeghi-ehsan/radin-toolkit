### توضیحات :
- علاوه بر Switch کامپوننت مکمل دیگری به نام SwitchOption نیز کدنویسی شده است که به عنوان children های Switch استفاده می‌شوند. یعنی هر Switch می‌تواند تعداد بیشماری Option داشته باشد. این کامپوننت هم به طور جدا SwitchOption و هم به صورت Switch.Option در دسترس و قابل استفاده است.

### props ها و قابلیت ها:

#### Switch props

- **name**: از آنجایی که این کامپوننت از input:radio استفاده می‌کند، نیاز به داشتن اسم input دارد. که از این اسم در id تمام label ها نیز استفاده می‌شود.
- **defaultValue**: از بین تمام گزینه‌هایی که نمایش داده می‌شود، یکی از آنها میتواند فعالی باشد، که کافیست فقط value آن را به defaultValue پاس دهیم.
- **onChange**: در این متد باید state ریکت مدیریت شود.
- **disabled**: با فعال کردن این گزینه، امکان انتخاب هیچ گزینه‌ای وجود نخواهد داشت.
- **vertical**: اگر true باشد switch button ها به صورت عمودی زیر هم قرار می گیرند.
- **innerClassName**: کلاسی که بر قسمت متحرک انتخاب گر تاثیر می گذارد.
- **outerClassName**: کلاسی که بر parent آپشن ها تاثیر می گذارد.
- **noLabelStyle**: برای استفاده از labelClassName و همین طور اپلای کردن استایل هایی بر فونت و پدینگ بدون استفاده از ! .
- **labelClassName**: کلاسی که بر خود آپشن ها تاثیر می گذارد.

#### SwitchOption props

- **children**: عنوان هر گزینه به صورت children پاس داده می‌شود.
- **name**: این مقدار مربوط به input می‌باشد که از طریق کامپوننت مادر یعنی Switch به SwitchOption پاس داده می‌شود.
- **value**: این مقدار مربوط به input می‌باشد که هنگام تغییر گزینه، به ورودی تابع onChange داده می‌شود.
- **checked**: همان value برای Switch است.
- **defaultValue**: مقدار دیفالت که می توان با آن مقدار اولیه ی switch را بدست آورد.
- **onChange**: تابعی که موقع تغییر اجرا میشود.
- **disabled**: با فعال کردن این گزینه، امکان انتخاب هیچ گزینه‌ای وجود نخواهد داشت.
- **isOptionDisabled**: با فعال کردن این گزینه، امکان انتخاب این گزینه وجود نخواهد داشت.
- **noLabelStyle**: برای استفاده از labelClassName و همین طور اپلای کردن استایل هایی بر فونت و پدینگ بدون استفاده از .
- **labelClassName**: کلاسی که بر خود آپشن ها تاثیر می گذارد.
